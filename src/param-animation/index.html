パラメータ・アニメーション ガイド VRChat Avatar

How-to / Unity + VRChat SDK3

マテリアル／シェーダーのパラメータを
アニメーションで動かす

VRChatアバターで、シェーダー（マテリアル）のプロパティを Expression Menu から動かすための汎用手順です。連続値は Radial Puppet、オン/オフは Toggle。AnimationClip・AnimatorController・Expression Parameters/Menu・Modular Avatar の組み方を、Unity Editor 上の操作として再現できる粒度でまとめています。

仕組みの全体像

やることは一本道です。メニュー操作が同期パラメータを変え、それがAnimatorレイヤーを通じてAnimationClipを再生し、クリップが Renderer 上の material.<プロパティ> を書き換えます。この鎖を1本ずつ用意していきます。

アバターへ組み込む部分（Animator・パラメータ・メニュー）は Modular Avatar でマージするのが本ガイドの推奨です。プレハブをアバター配下にドラッグするだけで済み、アバター側の既存メニュー／パラメータと衝突せず、複数アバターに使い回せます（手動で組む方法も後述）。

この手順が使える対象

アニメーション可能なマテリアルの float / color プロパティを持つシェーダーなら、製品を問わず同じ手順で動かせます。本ガイドは DeepFrostIce の実装を具体例として引用していますが、ColdAir や FrozeNade など他の製品でも、プロパティ名とレンジを読み替えるだけで同じように組めます。

連続値か、オン/オフか

最初に「そのパラメータをどう操作させたいか」を決めます。ここで メニューUI と クリップの形 がほぼ決まります。

操作したいものごとの組み方の対応表
動かしたいもの値の性質メニューUIAnimatorレイヤークリップ
0→1 や 0→5 の連続変化
（凍結量・融解量・吸収濃度・ディゾルブ量 など）		 Float Radial Puppet Motion Time（1ステート） from→to の線形カーブ1本
機能のオン/オフ
（トップスノー・スパークル など）		 Bool Toggle Off / On（2ステート） 定数カーブ2本
オブジェクトごとの表示/非表示 Bool Toggle Off / On（2ステート） m_IsActive 定数2本

連続値の Radial Puppet は「押している間だけリング状のスライダーが出て 0〜1 を回す」UI、Toggle は「押すたびに切り替わる」UIです。表示/非表示だけはマテリアルではなくオブジェクトの有効状態を動かします（後述）。

手順1クリップを作る

Animation ウィンドウで、対象は必ず Renderer が乗っている GameObject を選びます。バインドのキーは material.<プロパティ名> です（Unity では「Mesh Renderer → Material.＿プロパティ」として一覧に出ます）。

連続値のクリップ（線形カーブ1本）

先頭に from、末尾に to のキーを打った線形カーブを1本だけ入れます。クリップの長さ自体は Motion Time で正規化されるため意味を持ちません（1秒で十分）。0→1 以外のレンジ（例：吸収は 0→5）は、パラメータではなくクリップの値レンジ側で表現します。

オン/オフのクリップ（定数2本）

Off と On で1本ずつ、それぞれ1フレームの定数クリップを作ります。Write Defaults を off で運用する場合、Off クリップでも必ずそのプロパティを off 値で打つのが要点です（打たないと値が戻りません）。

表示/非表示のクリップ（m_IsActive）

オブジェクトごと消したい場合は、マテリアルではなく 子 GameObject の有効状態（m_IsActive）を動かします。ポイントは、Animator が乗るルートではなくその子（Renderer 側）を対象にすること。子を対象にすれば、非表示にしても Animator まで止まりません。

手順2Animatorレイヤーを組む

FX 用の AnimatorController を用意し、パラメータごとに1レイヤーを足します。マージ先に無駄が残らないよう、自動生成される空の Base Layer は削除しておきます。各レイヤーは Weight = 1、Write Defaults = off で統一します。

パラメータを宣言する

Animator の Parameters に、連続値は Float、オン/オフは Bool を追加します。名前は後述の同期パラメータ名と一致させます（食い違うと動きません）。

Motion Time レイヤー（Radial Puppet 用）

1ステートだけのレイヤーを作り、その State に 0→1 クリップを置きます。State のインスペクタで Motion Time を有効化し、対応する Float パラメータを割り当てます。すると 0〜1 のパラメータがクリップの正規化時間へ写像され、結果としてクリップの from→to（0→5 なども含む）へ写ります。

Off/On トグルレイヤー

Off / On の2ステートを作り、既定ステートは Off。Bool パラメータで相互遷移させます。Off→On は「If（bool = true）」、On→Off は「IfNot（bool = false）」。どちらも Has Exit Time = off、遷移時間は 0.1 秒程度（即応させたいなら 0）にします。

Write Defaults の統一

FX 全体で Write Defaults の on/off を混在させないのが鉄則です。本ガイドは off で統一する前提なので、各ステートのクリップに「そのステートで欲しい値」を明示的に入れています。Modular Avatar を使う場合は Match Avatar Write Defaults がアバター側の流儀に自動で合わせてくれます。

手順3同期パラメータとメニュー

Animator を動かすための 同期パラメータ と、それを触るための Expression Menu を用意します。

Expression Parameters

  • 連続値：Float、Saved 有効、既定値 0。
  • オン/オフ：Bool、Saved 有効、既定値は初期状態に合わせる（例：トップスノー／スパークル／表示は既定 ON）。
  • 名前は Animator のパラメータ名と完全一致させます。

Expression Menu

  • Radial Puppet：コントロール自身の Parameter は空のまま、Rotation（サブパラメータ）に Float 名を割り当てます。
  • Toggle：コントロールの Parameter に Bool 名を割り当て、Value = 1。
  • これら（例では7項目）をルートに直挿しせず、サブメニュー1個にまとめて、ルートにはそのサブメニューへのコントロールを1つだけ置きます。

手順4Modular Avatarで組む（推奨）

ここまでの Controller・Parameters・Menu を、プレハブのルートに載せた4つの MA コンポーネントでアバターへマージします。プレハブをアバター配下にドラッグするだけで完了し、既存の FX／メニュー／パラメータを壊しません。

  1. Animator（一時置き）

    プレハブのルートに Animator を付け、Controller に上で作った FX 用 Controller を指定します。これはマージ用の受け渡しで、下の Merge Animator がマージ後に自動で外します。

  2. MA Merge Animator

    • Merge Animator To = 作った Controller
    • Layer Type = FX
    • Delete Attached Animator = ON（上の一時 Animator を掃除）
    • Path Mode = Relative（クリップのパスをプレハブ内の Renderer に解決）
    • Match Avatar Write Defaults = ON

  3. MA Parameters

    同期パラメータをアバターの Expression Parameters へ登録します。Float は Saved・既定 0、Bool は Saved・明示的な既定値（既定 ON にしたいものは明示的に ON）を指定します。

  4. MA Menu Installer

    Menu to Append に「SubMenu を1つ持つルートメニュー」を指定します。アバターのメニューへその SubMenu が append され、ルートを埋めません。

MAの利点

Controller・Parameters・Menu をアバター本体に手作業でコピーしないので、アバター側の既存設定と衝突しない・複数アバターに使い回せる・更新差し替えが楽という利点があります。Renderer とマテリアルは同じプレハブ内に置き、Relative パスで解決させます。

MAを使わない手動組み込み

Modular Avatar を使わない場合は、同じ中身をアバター本体へ直接足します。手数が増え、衝突にも注意が要りますが仕組みは同じです。

  • FX Controller：手順2で作ったレイヤー（Motion Time／トグル）をアバターの FX AnimatorController へ複製。クリップのバインドパスが、アバター配下での Renderer のパスと一致するようパスを合わせ直します。
  • Expression Parameters：Float（Saved・0）／Bool（Saved・明示既定）を手動登録。
  • Expression Menu：SubMenu を作って Radial／Toggle コントロールを入れ、アバターのメニューへその SubMenu を追加。
  • Write Defaults はアバターの FX 全体の流儀に合わせて統一します（混在させない）。

実例：DeepFrostIce の対応表

本ガイドの下敷きにした DeepFrostIce の実装では、7項目を次のように割り当てています。他製品でも「メニュー項目 → 対象プロパティ → レンジ」を読み替えれば同じ構造で組めます。

つまずきやすい注意点

動かないとき・意図せぬ挙動のとき、まずここを疑ってください。

Write Defaults の混在

FX 内で on/off が混ざると値が戻らない・効かないの温床に。全体で統一。off なら各ステートで欲しい値を必ず明示します。

Motion Time のレンジ

Radial は必ず 0〜1。0→5 のような幅はクリップ側の値レンジで表現します。パラメータを 0〜5 にしようとしないこと。

ルートメニューの圧迫

ルートは8枠しかありません。項目はSubMenu にまとめて、ルートにはその入口を1つだけ置きます。

同期パラメータのコスト

同期は 256bit の枠を共有（Float=8bit / Bool=1bit）。必要な分だけにし、ローカルで十分なものは非同期も検討します。

名前の食い違い

Animator パラメータ／Expression Parameters／メニューの名前は完全一致。1文字違うだけで無反応になります。

表示切替の対象

オブジェクト非表示は Animator が乗る子（Renderer 側）の m_IsActive を動かします。ルートを消すと Animator まで止まります。
